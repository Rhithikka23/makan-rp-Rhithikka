Makan@RP System Architecture

  • Frontend built using HTML, CSS, and JavaScript
  • Botpress Cloud handles chatbot logic and NLP processing
  • User queries are sent securely to the chatbot backend
  • Responses are generated using predefined conversational flows
  • System design allows scalability and easy updates

Dataset Structure & Data Sources

  • Food stall names and campus locations
  • Menu items with prices and dietary labels
  • Operating hours and availability information
  • Frequently asked questions from students
  • Data manually curated and validated by the project team

User Interaction Flow

  1. User opens the Chat page
  2. User asks a food-related question
  3. Chatbot analyses intent and keywords
  4. Relevant response is generated
  5. User may continue or refine the query

FAQs & Troubleshooting

  • Chatbot not responding: Refresh the page
  • Incorrect recommendation: Rephrase the question
  • Outdated menu: Dataset updates are ongoing
  • Browser issues: Use latest Chrome or Edge
